# Agentic RAG 设计书文档包

本目录用于指导 **Atlas Agentic RAG Copilot** 项目的设计、拆解与实现，可直接与 Codex 协作使用。

## 文档结构

1. `01_系统架构设计说明书.md`
   - 项目定位、总体目标、系统分层、主链路、技术选型
   - 用于统一项目方向与对外讲解

2. `02_详细模块设计说明书.md`
   - 模块职责、边界、图节点、状态模型、模块间调用关系
   - 用于指导后端领域建模与前后端协同

3. `03_数据库表结构详细设计说明书.md`
   - 数据库 schema、表字段、索引、约束、迁移顺序
   - 用于指导建表、迁移、Repository 层实现

4. `04_前后端接口与页面原型说明书.md`
   - 页面划分、组件树、关键交互、API 设计、SSE 事件协议
   - 用于指导前端实现、接口联调与演示设计

5. `05_开发路线图与任务拆解.md`
   - 里程碑、阶段目标、任务优先级、风险点、Codex 协作建议
   - 用于指导分阶段推进

6. `06_部署、安全、评测与运维说明书.md`
   - 部署方案、安全策略、评测体系、观测与日志、运维建议
   - 用于指导上线、演示与质量闭环

7. `99_参考资料.md`
   - 近一到两年与本项目高度相关的官方文档与参考资料

## 推荐使用方式

### 方案一：你主导，Codex 实现
先把以下顺序发送给 Codex：
1. `01_系统架构设计说明书.md`
2. `02_详细模块设计说明书.md`
3. `03_数据库表结构详细设计说明书.md`

再要求 Codex：
- 先建立项目目录结构
- 再建立数据模型和迁移
- 再实现基础 RAG 主链路
- 最后接入 Agent、MCP、前端页面

### 方案二：按阶段逐步协作
- 第 1 阶段：数据库 + 文档上传 + 切分 + embedding
- 第 2 阶段：检索 + rerank + citations
- 第 3 阶段：LangGraph + Tool + HITL
- 第 4 阶段：MCP + 记忆 + 前端增强 + 评测

## 建议实现顺序

- 先完成 `03_数据库表结构详细设计说明书.md`
- 再完成 `04_前后端接口与页面原型说明书.md`
- 再按 `05_开发路线图与任务拆解.md` 分阶段推进

## 给 Codex 的执行总原则

- 严格遵守文档中的模块边界，不要随意耦合
- 代码风格统一，目录结构清晰
- 每个模块写简洁注释，说明用途与边界
- 优先做“能跑通的主链路”，再做高级能力
- 任何新增设计都应尽量兼容本设计书，不要推翻式重写
