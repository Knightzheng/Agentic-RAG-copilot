## 1. 文档目标

本文档在总体架构设计说明书基础上，继续下钻到：

- 模块职责
- 模块输入输出
- LangGraph 状态字段

- 模块间调用关系

- 前后端对应关系

目标是把系统拆到“可以直接分模块实现”的粒度。

## 3. 前端应用模块

### 3.1 职责
- 用户操作入口
- 页面状态维护
- 流式结果展示
- 执行步骤展示
- 引用展示
- 审批交互
- 文档与记忆管理

### 3.2 页面拆分
- Chat Workspace
- Knowledge Base
- Trace Center
- Memory Center
- MCP Hub

### 3.3 前端状态分层建议

```ts
type AppState = {
  auth: AuthState;
  workspace: WorkspaceState;
  threads: ThreadState;
  currentRun: RunState | null;
  kb: KnowledgeBaseState;
  memory: MemoryState;
  mcp: MCPState;
  ui: UIState;
};
```

### 3.4 核心交互模式
- 聊天：`POST /api/chat` + SSE
- 上传：`POST /api/documents/upload` + 轮询 job

## 7. 文档解析模块

### 7.1 职责

### 7.2 输出内容
- `page_no`
- `block_order`
- `block_type`
- `section_path`
- `raw_text`

### 7.3 block_type
- `title`
- `paragraph`
- `list`
- `table`
- `code`
- `quote`

---

## 8. Chunk 切分与上下文化模块

### 8.1 职责
从 block 构建 parent-child chunk，并做 contextualization。

### 8.2 Parent Chunk
用于：
- 提供大上下文
- small-to-big 扩展
- 较完整章节信息保留

### 8.3 Child Chunk
用于：
- 精准召回
- rerank 输入
- 最终引用绑定

### 8.4 contextualization 内容
- 文档标题
- 章节路径
- 当前段落主题
- 必要背景信息

---

class AtlasAgentState(TypedDict):

## 13. RAG 子图模块

### 13.1 节点列表
1. `normalize_query`
2. `rewrite_query_if_needed`
3. `retrieve_candidates`
4. `merge_and_expand`
5. `rerank_candidates`
6. `grade_evidence`
7. `generate_grounded_answer`
8. `bind_citations`
9. `return_rag_result`

### 13.2 生成约束
回答生成阶段只允许使用：
- evidence chunks
- 必要的 recalled memories
- 少量最近消息上下文

---

## 14. Tool Execution 子图模块

### 14.1 节点列表
1. `plan_tool_steps`
2. `select_tool`
3. `risk_check`
4. `interrupt_if_needed`
5. `execute_tool`
6. `observe_result`
7. `reflect_and_decide`
8. `return_tool_result`

### 14.2 ReAct 约束
- 最大工具步数：4
- 最大连续失败：2
- 高风险动作必须审批
- 工具返回空值禁止无限重试

### 14.3 工具分类
- 信息型工具
- 分析型工具
- 系统型工具
- MCP 工具

## 18. 审批与中断模块

### 18.1 触发条件
- 写数据库
- 删除资源
- 覆盖写文件
- 触发外部副作用
- 高风险 MCP 工具

### 18.2 流程
1. 节点检测到高风险动作
2. 构造审批 payload
3. 触发 interrupt
4. 前端展示审批 UI
5. 用户 approve / reject / modify
6. 节点 resume
7. 执行后续逻辑

## 19. 日志、观测与评测模块

### 19.1 记录维度
- run 级
- step 级
- retrieval 级
- tool 级

### 19.2 评测维度
- 检索：Recall@K / MRR / nDCG
- 生成：Faithfulness / Relevance / Completeness

- `viewer`
