# 《Agentic RAG 数据库表结构详细设计说明书》

**文档版本**：V1.0  
**文档定位**：数据库蓝图 / 建表与迁移依据 / Repository 层实现依据

---

## 1. 文档目标

本文档用于确定 Atlas Agentic RAG Copilot 的数据库落地结构，明确：

- schema 划分
- 表清单
- 字段定义
- 主键与外键
- 唯一约束
- 索引设计
- 状态字段
- 向量索引策略
- 迁移顺序

---

## 2. 存储选型与总体原则

本项目数据库底座采用：
- PostgreSQL
- pgvector

理由：
- 业务数据、检索数据、记忆、日志、审批统一存储
- 便于 join、过滤、审计
- 减少早期多系统耦合
- 支持向量检索与全文检索并存

### 原则
1. 统一底座，分域建模
2. 热表与冷表分离
3. 所有状态显式化
4. 所有可追踪链路必须可回溯
5. 向量数据与关系数据必须强关联

---

## 3. Schema 划分

建议采用单库多 schema：
- `app_core`
- `app_kb`
- `app_agent`
- `app_memory`
- `app_obs`
- `app_mcp`

---

## 4. 通用约定

### 4.1 主键
全部使用 `uuid`

### 4.2 时间字段
统一使用 `timestamptz`

### 4.3 软删除
只对用户资源做软删除：
- workspaces
- documents
- threads
- memories
- mcp_servers

### 4.4 状态字段
统一命名：
- `status`
- `phase`
- `result_status`

## 12. 结论

数据库设计的核心目标是让系统做到：

- 可建表
- 可迁移
- 可查询
- 可追踪
- 可审计
- 可恢复

最重要的原则：

**关系数据、向量数据、运行数据必须形成统一、可追踪的闭环。**
