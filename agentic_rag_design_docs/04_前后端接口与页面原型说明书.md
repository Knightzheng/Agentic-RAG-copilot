## 2. 前端产品结构

系统建议采用五页式主结构：

1. Chat Workspace
2. Knowledge Base
3. Trace Center
4. Memory Center
5. MCP Hub

全局布局建议：
- 左侧：导航栏
- 顶部：工作区切换 + 用户菜单 + 全局状态
- 中央：具体页面内容
- 全局：Toast / Modal / Approval Dialog / Error Banner

## 3. Chat Workspace 页面

### 3.1 页面目标
主入口页面，负责：
- 发起对话
- 展示模型回答
- 展示执行步骤
- 展示引用
- 展示工具调用
- 接收审批请求

### 3.2 页面布局
- 左栏：会话列表
- 中栏：消息列表 + 输入区
- 右栏：步骤、引用、工具、审批

### 3.3 组件树建议

```text
ChatWorkspacePage
├─ ThreadSidebar
├─ ChatMainPanel
│  ├─ MessageList
│  ├─ Composer
│  └─ RunStatusBar
└─ RightInspector
   ├─ StepTimeline
   ├─ CitationPanel
   ├─ ToolPanel
   └─ ApprovalPanel
```

### 3.4 核心交互
1. 用户输入问题
2. 调用 `POST /api/chat`
3. 返回 `run_id`
4. 订阅 SSE
5. 展示 token / step / citation / approval

## 8. 核心 API 设计

### 8.1 聊天与线程 API

#### `POST /api/chat`
请求体：
```json
{
  "thread_id": "optional-thread-id",
  "workspace_id": "ws_xxx",
  "message": "请总结这个制度中的报销流程",
  "attachments": ["doc_xxx"],
  "mode": "auto",
  "metadata": {}
}
```

响应体：
```json
{
  "run_id": "run_xxx",
  "thread_id": "th_xxx",
  "status": "running"
}
```

#### `GET /api/threads`
获取线程列表。

#### `GET /api/threads/{thread_id}`
获取线程详情与消息列表。

#### `POST /api/threads/{thread_id}/resume`
恢复被中断或等待审批的线程。

### 8.2 文档 API
- `POST /api/documents/upload`
- `GET /api/documents`
- `GET /api/documents/{document_id}`
- `GET /api/documents/{document_id}/chunks`
- `POST /api/documents/{document_id}/reindex`

### 8.3 Trace API
- `GET /api/runs/{run_id}`
- `GET /api/runs/{run_id}/steps`
- `GET /api/runs/{run_id}/trace`

### 8.4 Memory API
- `GET /api/memory`
- `PATCH /api/memory/{memory_id}`
- `DELETE /api/memory/{memory_id}`
- `POST /api/memory/{memory_id}/pin`

### 8.5 MCP API
- `GET /api/mcp/servers`
- `POST /api/mcp/connect`
- `POST /api/mcp/{server_id}/sync-tools`
- `GET /api/mcp/{server_id}/tools`
- `POST /api/mcp/{server_id}/health-check`

### 8.6 审批 API
- `POST /api/approvals/{approval_id}/approve`
- `POST /api/approvals/{approval_id}/reject`
- `POST /api/approvals/{approval_id}/modify`

### 8.7 反馈 API
- `POST /api/feedback`

## 9. SSE / 流式事件协议

建议使用以下事件：

- `run.started`
- `step.started`
- `step.completed`
- `token.delta`
- `citation.update`
- `tool.started`
- `tool.completed`
- `approval.required`
- `run.completed`
- `run.failed`

### 示例：`approval.required`
```json
{
  "run_id": "run_xxx",
  "approval_id": "appr_xxx",
  "approval_type": "sql_write",
  "payload": {
    "statement_preview": "DELETE FROM ..."
  }
}
```

## 10. 错误与空态设计

### 10.1 错误提示分级
- 轻提示：网络轻微波动、自动重试成功
- 中断提示：SSE 断开、文档处理中、等待审批
- 阻断提示：权限不足、上传失败、run 启动失败

### 10.2 空态
- 没有文档：提示上传第一个文档
- 没有记忆：提示系统尚未沉淀长期记忆
- 没有 MCP：提示添加 MCP Server
- 没有 trace：提示先运行一次任务

6. run trace 查询
