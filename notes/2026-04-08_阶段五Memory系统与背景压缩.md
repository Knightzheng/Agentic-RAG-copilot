# 2026-04-08 阶段五 Memory 系统与背景压缩

## 本轮目标
- 在现有多轮会话基础上落地完整的长期记忆系统
- 先把线程历史压缩成可复用的工作记忆摘要，再接入 semantic / episodic / procedural memory
- 提供后端 CRUD、图编排 recall、以及前端 Memory Center

## 已完成
- 新增 `app_memory` schema 及 6 张表：
  - `memory_namespaces`
  - `semantic_memories`
  - `semantic_memory_embeddings`
  - `episodic_memories`
  - `episodic_memory_embeddings`
  - `procedural_memories`
- `ChatService` 在每轮完成后会：
  - 自动写入显式 semantic memory
  - 自动写入 compact episodic memory
  - 对线程早期消息做背景压缩，并写回 `thread.metadata_json.thread_summary`
- `AtlasSupervisorGraphService` 新增：
  - `recall_long_term_memory` 节点
  - `thread_summary` + `recalled_memories` 注入 direct / grounded answer prompt
  - `load_thread_context` 只携带最近窗口消息，并依赖压缩摘要承接更早背景
- 新增 Memory API：
  - `GET /api/memory`
  - `POST /api/memory`
  - `PATCH /api/memory/{memory_id}`
  - `DELETE /api/memory/{memory_id}`
  - `POST /api/memory/{memory_id}/pin`
- 前端 `Memory Center` 已可：
  - 筛选 memory type
  - 关键词搜索
  - pinned-only 过滤
  - 创建 / 编辑 / pin / 删除 memory
  - 从 memory 跳转 source run trace
- 当前线程面板新增 `Compressed Background` 展示，便于人工确认背景摘要是否生效
