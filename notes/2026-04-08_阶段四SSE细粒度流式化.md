# 2026-04-08 阶段四补充：SSE 细粒度流式化

## 本轮目标

- 将 `chat/stream` 从聚合式 `rag` 事件改为细粒度节点流
- 保持同步 `/api/chat` 和 Trace 落库逻辑不回归
- 让前端直接看到 RAG 内部步骤

## 已完成内容

### 1. 主图内联 RAG 节点

- `AtlasSupervisorGraphService` 不再通过单独的 `_rag_graph.invoke(...)` 聚合执行 RAG
- 现在将以下节点直接挂到主图中：
  - `invoke_rag_subgraph`
  - `rag.normalize_query`
  - `rag.retrieve_candidates`
  - `rag.grade_evidence`
  - `rag.generate_grounded_answer`
- 这样 `graph.stream(..., stream_mode="updates")` 会逐节点输出更新事件

### 2. SSE step 输出摘要化

- 为 `RunTraceRecorder` 增加公开的 `summarize_update(...)`
- `POST /api/chat/stream` 的 `step` 事件现在先做 JSON-safe 摘要化再发给前端
- 避免 SSE 里直接暴露复杂 ORM / dataclass 对象

### 3. 前端流式进度增强

- `Chat Workspace` 现在能按顺序看到：
  - `load_thread_context`
  - `classify_request`
  - `invoke_rag_subgraph`
  - `rag.normalize_query`
  - `rag.retrieve_candidates`
  - `rag.grade_evidence`
  - `rag.generate_grounded_answer`
  - `compose_final_answer`
  - `finish`
- 前端对 step 输出增加了更易读的摘要逻辑，例如：
  - `query=...`
  - `retrieved=...`
  - `evidence=...`
  - `citations=...`

## 当前边界

- 现在的流式仍然是节点级，不是 token 级
- `rag.generate_grounded_answer` 内部的模型生成过程还没有继续拆成 token stream
- 前端当前只展示 step 摘要，没有展开每一步的完整输出 diff

## 下一步建议

1. 开始做真正的 token streaming
2. 让 `rag.generate_grounded_answer` 在生成阶段持续向前端推送增量文本
3. 再往后接入 Memory 节点，把多轮会话状态也纳入图编排
