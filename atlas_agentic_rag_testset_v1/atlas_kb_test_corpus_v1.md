# Atlas Agentic RAG 测试知识库文档（V1）

> 文档性质：为 Agentic RAG 项目联调、评测与演示而编写的**合成企业知识库文档**。  
> 文档版本：V1.0  
> 生效日期：2026-02-01  
> 适用对象：售前顾问、实施工程师、平台管理员、运维值班人员、客户成功经理  
> 说明：除特别标注“旧版规则”“历史说明”的条目外，本文档中的规则均以 V1.0 为准。

## 1. 平台概览

Atlas 是一套面向企业知识问答、流程检索、文档协作与外部工具接入的智能知识平台。平台由五个一级模块组成：

1. **Knowledge Hub**：文档接入、切分、索引、检索与引用展示。
2. **Agent Studio**：图结构编排、ReAct 工具使用、审批中断与恢复。
3. **Memory Center**：工作记忆、情节记忆、语义记忆、程序性记忆管理。
4. **MCP Hub**：外部 MCP Server 连接、工具注册、权限分级与调用审计。
5. **Ops Console**：运行追踪、评测看板、告警与审计导出。

平台支持三种部署形态：**Cloud SaaS、Dedicated Cloud、On-Prem**。  
Cloud SaaS 由平台方统一运维；Dedicated Cloud 为客户独享云环境；On-Prem 由客户在内网自建。

## 2. 产品版本与套餐

Atlas 目前提供三个商业套餐：**Starter、Business、Enterprise**。

### 2.1 Starter
- 最大工作区数：3
- 每工作区最大成员数：20
- 单文件大小上限：50 MB
- 每月文档上传额度：2,000 份
- 每月对话额度：50,000 次
- 默认保存最近 90 天运行日志
- 不支持私有化部署
- 不支持自定义审批流
- 最多连接 2 个 MCP Server

### 2.2 Business
- 最大工作区数：20
- 每工作区最大成员数：200
- 单文件大小上限：200 MB
- 每月文档上传额度：20,000 份
- 每月对话额度：500,000 次
- 默认保存最近 180 天运行日志
- 支持 Dedicated Cloud
- 支持自定义审批流
- 最多连接 20 个 MCP Server

### 2.3 Enterprise
- 工作区数不限
- 成员数按合同约定
- 单文件大小上限：500 MB
- 文档上传与对话额度按资源包购买
- 默认保存最近 365 天运行日志
- 支持 Dedicated Cloud 与 On-Prem
- 支持专属 SSO、审计导出、私有模型接入、专线网络
- MCP Server 数量不设硬性上限，但建议单工作区不超过 80 个

### 2.4 计费补充规则
- 文档上传额度按“成功入库的文档版本”计算，解析失败不计费。
- 同一文档重复上传且内容哈希完全一致时，不重复计入上传额度。
- 对话额度按“成功创建 run 的请求”计算，被权限系统拦截的请求不计入额度。
- 从 2026-02-01 起，Starter 套餐的 MCP 连接上限由 **1 个**提升至 **2 个**；旧销售资料中的“1 个”已失效。

## 3. 用户角色与权限

平台有四类内置角色：**Owner、Admin、Editor、Viewer**。

### 3.1 Owner
- 可删除工作区
- 可转移所有权
- 可管理计费与套餐
- 可创建、禁用、删除 MCP 连接
- 可查看所有审计日志
- 可配置保留策略

### 3.2 Admin
- 可管理成员
- 可上传、删除文档
- 可修改文档标签
- 可配置审批规则
- 可查看运行追踪和评测结果
- 不可转移工作区所有权
- 不可查看计费合同附件

### 3.3 Editor
- 可上传文档
- 可发起对话
- 可编辑工作区提示词模板
- 可手动 pin/unpin 记忆
- 可查看自己发起的运行记录
- 不可删除工作区
- 不可修改成员权限

### 3.4 Viewer
- 可发起问答
- 可查看共享文档
- 可查看公开记忆卡片
- 不可上传文档
- 不可修改设置
- 不可调用被标记为 high risk 的工具

### 3.5 权限例外
- 即使是 Owner，也不能绕过“高风险外部写操作”的审批；Owner 只能成为审批人之一，不能取消审批机制本身。
- Viewer 在 Business 和 Enterprise 套餐中都可以使用问答功能，但 Starter 套餐下 Viewer 默认不可导出引用内容。

## 4. 工作区与资源配额

### 4.1 工作区默认配额
每个工作区有以下默认资源配额：

- 文档总数上限：100,000
- chunk 总数软上限：20,000,000
- 长期记忆条目软上限：2,000,000
- 每线程消息软上限：5,000 条
- 单次 run 最大工具步数：8
- 单次 run 最大并行 worker 数：6

### 4.2 软硬限制
- “软上限”达到时，系统会发出告警，但不会立即拒绝写入。
- “硬上限”达到时，系统会拒绝新写入。
- Business 套餐的默认 chunk 总数硬上限是 30,000,000；Enterprise 套餐可按合同调整。

### 4.3 资源清理
- 归档线程默认 30 天后进入冷存储。
- 运行日志冷存储后仍可检索摘要，但不能直接查看完整 step payload。
- 被删除文档的原文件会先进入 7 天回收站；回收站清空后才会真正删除对象存储中的文件。

## 5. 文档接入规则

### 5.1 支持格式
目前支持：
- PDF
- DOCX
- PPTX
- Markdown
- TXT
- HTML

暂不支持：
- 扫描件图片直接 OCR 入库
- 受密码保护的压缩包
- 可执行文件
- 纯图片文件作为知识库主文档

### 5.2 入库流程
标准入库流程为：
1. 上传文件
2. 计算哈希
3. 去重检测
4. 文档解析
5. block 抽取
6. chunk 切分
7. contextualization
8. embedding
9. 检索索引写入
10. 状态回写

### 5.3 文档状态
- uploaded
- processing
- ready
- failed
- archived

### 5.4 去重规则
如果同一工作区中存在文件哈希完全一致、且处于未删除状态的文档，系统默认不创建新文档，只创建一个“版本引用事件”。  
如果用户强制要求重建索引，可以创建新版本，但不会创建新的逻辑文档主记录。

### 5.5 表格处理
- 表格优先按“表头 + 行记录”方式结构化。
- 表格抽取失败时，降级为文本块。
- 表格不会和前后正文混切到同一个 child chunk 中。

## 6. Chunk 与索引策略

### 6.1 默认切分策略
- parent chunk：1200~1600 tokens
- child chunk：250~350 tokens
- overlap：60 tokens

### 6.2 特殊内容策略
- 标题不单独回答问题，但会进入 chunk 的 context prefix。
- 表格单独切块。
- 代码块默认保留原缩进。
- 超长列表按二级标题分段切分。

### 6.3 Contextualization 规则
child chunk 在 embedding 前会补充：
- 文档标题
- 章节路径
- 邻近主题摘要
- 关键实体标签

### 6.4 检索策略
默认采用 **Hybrid Retrieval**：
- dense topK = 20
- lexical topK = 20
- merge 后候选 = 30~40
- rerank 后最终证据 = 6~8

### 6.5 重排序策略
- 若问题包含明确年份、版本号、价格、阈值，时间或数值匹配权重上调。
- 若问题是“谁可以”“是否允许”类权限问题，标题命中与规则型段落权重上调。
- 若问题来自多轮对话中的追问，最近一轮用户消息和 thread summary 会参与 query rewrite。

## 7. 记忆系统规则

平台采用四层记忆：

### 7.1 Working Memory
- 范围：当前 thread
- 内容：最近消息、当前证据、工具结果、中间变量
- 默认保留：最近 40 条消息窗口
- 超出后触发 thread summary 压缩

### 7.2 Episodic Memory
- 记录过去发生过的任务过程
- 示例：某次复杂报表分析的步骤摘要、某次工具失败原因
- 默认只保留最近 180 天的未 pin episode
- 被 pin 的 episode 不受 180 天规则影响

### 7.3 Semantic Memory
- 记录稳定事实与用户偏好
- 示例：用户常用术语、项目别名、部门简称、回答风格偏好
- 默认长期保留，除非用户删除或管理员执行清理策略

### 7.4 Procedural Memory
- 记录处理策略
- 示例：财报类问题优先查表格、合同类问题优先返回保守答案并加强引用
- 默认只有 Admin 及以上角色可修改

### 7.5 记忆写入规则
- 用户显式说“记住”时，优先写 semantic memory
- 高价值任务流程可异步提炼为 episodic memory
- 系统级规则不从普通对话中自动写入 procedural memory，必须经过审核或模板发布

## 8. MCP 接入与工具策略

### 8.1 支持的 MCP 传输方式
- stdio
- Streamable HTTP

### 8.2 MCP 工具风险分级
- low：只读查询、无外部副作用
- medium：读取敏感系统、访问受限数据
- high：写数据库、删文件、发 webhook、修改外部系统状态

### 8.3 默认审批规则
- low：默认免审批
- medium：Business 与 Enterprise 默认审批可选，Starter 默认必须审批
- high：所有套餐都必须审批

### 8.4 MCP 数量建议
- Starter：最多 2 个
- Business：最多 20 个
- Enterprise：不设硬上限，但建议单工作区不超过 80 个，以降低工具选择噪声

### 8.5 工具调用约束
- 同一 run 默认最多调用 8 个工具步骤
- 连续失败 2 次后必须反思或停止
- 高风险工具即使参数未真正执行，也要留下审批审计记录

## 9. 审批与中断规则

### 9.1 触发审批的场景
- 写数据库
- 删除或覆盖文件
- 外部系统状态修改
- 调用 high risk MCP 工具
- 导出包含敏感字段的完整审计日志

### 9.2 审批状态
- pending
- approved
- rejected
- modified
- expired

### 9.3 审批时限
- 默认审批有效期：24 小时
- 超过有效期未处理则自动变为 expired
- expired 的审批不会自动重试，需要用户重新发起请求

### 9.4 审批人与越权规则
- Admin 可以审批 medium risk
- Owner 可以审批 medium risk 和 high risk
- 如果工作区启用了“双人审批”，则 high risk 至少需要 2 名审批人，其中 1 名必须是 Owner

## 10. 运行追踪与日志

### 10.1 运行状态
- running
- waiting_approval
- completed
- failed
- cancelled

### 10.2 结果状态
- success
- partial
- insufficient
- rejected
- error

### 10.3 运行日志保留
- Starter：90 天
- Business：180 天
- Enterprise：365 天

### 10.4 冷存储规则
- 归档 30 天后的线程，其详细 step payload 会转入冷存储
- 转冷后仍保留 run 摘要、耗时、token 使用、引用信息
- 如果客户开启“严格审计模式”，则 step payload 冷存储保留期从默认 180 天提升到 365 天

### 10.5 引用规则
- 默认每个回答句子最多绑定 2 条引用
- 若系统判断证据冲突，可在同一句绑定 3 条引用，但必须在答案中明确说明“存在冲突”

## 15. 安全与合规策略

### 15.1 身份认证
- Cloud SaaS 默认支持邮箱登录与 SSO
- Enterprise 支持 SAML 和 OIDC
- On-Prem 的认证方式由客户接入自身 IdP

### 15.2 数据加密
- 传输中：TLS 1.2+
- 存储中：对象存储与数据库磁盘加密
- Enterprise 可选客户自管密钥

### 15.3 审计日志
以下操作必须写审计日志：
- 成员权限变更
- 文档删除
- 审批通过/拒绝
- MCP server 新增/删除
- 导出完整审计记录
- 修改保留策略

### 15.4 数据删除
- 普通删除：进入 7 天回收站
- 强制删除：仅 Owner 可执行，且需二次确认
- 如果工作区启用了“严格审计模式”，强制删除不会删除已有审计日志，只会对敏感内容做脱敏保留

## 16. 评测策略

### 16.1 离线评测
使用以下指标：
- Recall@K
- MRR
- nDCG
- Faithfulness
- Citation Precision
- Answer Relevance

### 16.2 在线评测
- run 成功率
- evidence insufficient 触发率
- tool success rate
- average latency
- feedback 正向率

### 16.3 验收建议阈值
对于标准知识问答场景，推荐以以下阈值作为初步验收门槛：
- Retrieval Recall@5 ≥ 0.85
- Citation Precision ≥ 0.90
- Answer Faithfulness ≥ 0.92
- Tool Success Rate ≥ 0.95
- Sev-1 故障月发生次数 ≤ 1

## 18. 术语表

- **Working Memory**：当前线程运行态短期上下文
- **Episodic Memory**：任务过程型长期记忆
- **Semantic Memory**：稳定事实型长期记忆
- **Procedural Memory**：策略规则型长期记忆
- **Hybrid Retrieval**：向量检索与词法检索混合召回
- **Rerank**：对候选证据做二次排序
- **MCP**：模型上下文协议，用于把外部工具或资源以统一方式接入模型应用
- **Step Payload**：单个图节点执行的输入输出详情
- **冷存储**：保留摘要与必要索引，但不提供高频直接访问的低成本存储层
