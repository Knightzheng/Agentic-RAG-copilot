# Atlas 测试知识库评测说明（V1）

## 1. 目标

这套数据不是学术 benchmark 的替代品，而是你自己的 Agentic RAG 项目的第一套“工程可控测试集”。

它更适合验证：
- 你的文档上传、切分、向量化链路是否正常
- 检索和重排序是否能命中正确段落
- 答案是否能正确引用
- 多轮追问是否能利用 thread context
- 面对信息不足的问题时，系统能否保守回答

## 2. 推荐评分字段

### 2.1 检索层
- hit@5：正确支撑章节是否出现在前 5 个证据中
- hit@10：正确支撑章节是否出现在前 10 个证据中
- section_recall：支撑章节召回率

### 2.2 生成层
- exact_match：标准答案是否精确命中
- semantic_match：表达不同但结论一致
- citation_ok：回答是否至少引用到正确章节
- faithfulness：回答是否包含文档中没有支持的信息

### 2.3 Agent 层
- context_follow_ok：追问时是否正确继承上文指代
- outdated_rule_avoid：是否优先采用新版规则
- insufficient_detection：证据不足题是否识别为信息不足

## 3. 判分建议

### easy
- 应接近 100% 命中
- 引用至少 1 条正确章节

### medium
- 允许表达差异
- 但结论必须正确
- 多跳问题至少命中 2 个相关章节中的关键一处

### hard / insufficient
- 重点看系统是否乱答
- 如果信息不足，允许输出“无法确定”“文档未说明”“不能据此推出”

## 4. 初步验收线

建议你在这个数据集上至少达到：

- easy 准确率 ≥ 95%
- medium 准确率 ≥ 85%
- citation_ok ≥ 90%
- insufficient_detection ≥ 90%
- follow-up 正确率 ≥ 85%
